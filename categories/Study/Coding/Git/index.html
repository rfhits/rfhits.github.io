01-git基本使用

大家看2020年以前的git教学视频的时候，

记得将 Git 默认分支从 master 修改为 main

或者将分支从master切换到main

随便写一些自己对git这个工具粗浅的认识。

第一次接触git

挺早就接触过这个东西，但是，觉得不好用。

那时对命令行还没那么熟悉，个人太依赖于“可视化”了。

以及对整个git结构上的迷茫，

这样导致的后果就是，往GitHub上传资料都是通过网页解决，根本没有什么clone、commit和push之类的。

后来，用了vscode，爽了不少，本地完，commit一下，再点击下左下角同步，完事。

要做的事只有写comment，Ctrl+Enter，点下鼠标，

这样不香嘛~

git是个版本控制的工具

后来，慢慢了解git，对ta有个大概的认识

各种区/库

init啥就不说了，git的精髓就在那几个仓库上：

工作区，暂存区和版本库。工作的流程就是工作区–>暂存区–>仓库

和暂存区有关的英语单词就是stage和index，

工作区就是自己能看到文件、文件夹，

add就是工作区–>暂存区，

commit就是暂存区–>仓库库。

知道了这些，GitHub就相当于一个网盘了，就是把本地的仓库同步到远程的仓库。

撤销和删除

对我个人而言，很少用到撤销和删除，大不了，工作区重做了再更新整个仓库。

什么rm、status、checkout、reset，头晕。

实际上，git监控着工作区、暂存区和仓库
